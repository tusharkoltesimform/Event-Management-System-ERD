This repository contains an **ER diagram and database design** for an **Event Management System**.
It models how events, attendees, venues, sessions, sponsors, tickets, and feedback interact with each other.

#  Key Features

* Central **Event** entity with budget and revenue tracking
* Support for **multiple sessions per event**
* Proper handling of **many-to-many relationships** using junction tables
* Venue booking and availability management
* Ticket sales with pricing, discounts, and sales channels
* Sponsor funding, agreements, and sponsorship levels
* Attendee feedback and ratings for events and sessions
